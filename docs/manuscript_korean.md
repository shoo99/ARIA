## 초록 (Abstract)

RNA-seq 전사체 분석은 품질 관리, 정렬, 정량화, 차등 발현 검정, 경로 분석, 생물학적 해석에 이르는 다단계 워크플로를 필요로 한다. nf-core/rnaseq와 같은 자동화 파이프라인이 이러한 단계를 재현 가능하게 실행하지만, 단계 사이의 핵심 의사결정 — 품질 지표 평가, 통계 방법 선택, 중간 결과에 기반한 분석 전략 적응, 생물학적 맥락에서의 결과 해석 — 은 여전히 전문 생물정보학자에게 의존하고 있다. 본 연구에서는 대규모 언어 모델(LLM)을 추론 엔진으로 활용하여 RNA-seq 분석의 의사결정 공간을 자율적으로 탐색하는 오픈소스 프레임워크 ARIA(Adaptive Reasoning for Integrated Analysis)를 제안한다. ARIA는 품질 평가, 전략 적응, 방법 선택, 결과 해석을 관장하는 8개의 의사결정 지점(Decision Points, DPs)을 구현하며, 규칙 기반 임계값과 LLM 기반 맥락적 추론을 결합한다. 4개의 공개 RNA-seq 데이터셋 — SEQC(GSE49712, 10,430 DEGs), Airway(GSE52778, paired design, 951 DEGs), Fmr1 KO(GSE180135, 398 DEGs), Pasilla(GSE18508, 혼합 라이브러리, 224 DEGs) — 에서 3개 종에 걸쳐 벤치마크를 수행하였다. ARIA는 paired 실험 설계를 정확히 인식하여 DEG 검출을 21–47% 향상시키고, 라이브러리 유형 등 기술적 공변량을 감지하며(+4–30% DEG 증가), DEG 수에 따라 분석 전략을 적응적으로 선택하고, DESeq2/edgeR/limma-voom 간 교차 검증(r > 0.99)을 수행한다. Airway에서 7개 dexamethasone 표적 유전자 전부(7/7), Fmr1 KO에서 FMRP 표적 9/11개, SEQC에서 뇌/암 마커 15/16개가 정확히 할당되었다. ARIA는 https://github.com/shoo99/ARIA 에서 자유롭게 이용할 수 있다.

**키워드:** RNA-seq, 차등 발현, 대규모 언어 모델, 자율 분석, 의사결정 인식 파이프라인, GSEA

## 1. 서론

### 1.1 전사체 분석의 의사결정 문제

Bulk RNA-seq는 생물학적 조건 간 전사체 전반의 유전자 발현 변화를 프로파일링하는 표준 방법이 되었다(Stark et al., 2019). 일반적인 분석은 원시 시퀀싱 데이터의 품질 평가, 참조 유전체에 대한 리드 정렬, 전사체 정량화, 정규화, 차등 발현(DE) 검정, 기능적 농축 분석, 생물학적 해석의 다단계 순차적 과정을 포함한다(Conesa et al., 2016). 각 단계는 확립된 도구를 사용하지만 — 정렬에 STAR(Dobin et al., 2013), 정량화에 Salmon(Patro et al., 2017), DE 검정에 DESeq2(Love et al., 2014) — 이러한 단계를 연결하는 의사결정은 비자명적이며 분석 결과에 유의한 영향을 미친다.

예를 들어, DE 분석에서 검출된 차등 발현 유전자(DEGs)의 수는 하류 분석 전략을 직접적으로 결정한다: 수백 개의 DEGs가 있는 데이터셋은 전통적인 과대 대표 분석(ORA)을 지원하지만, DEGs가 적은 데이터셋은 임의의 유의성 기준이 아닌 전체 유전자 순위 목록을 활용하는 유전자 세트 농축 분석(GSEA; Subramanian et al., 2005)이 필요하다. 마찬가지로, 배치 효과, paired 실험 설계, 예상치 못한 세포 유형 시그니처의 존재는 각각 경험 많은 생물정보학자만이 인식하고 대응할 수 있는 특수한 분석적 대응을 요구한다.

### 1.2 기존 자동화의 한계

현재의 워크플로 관리자와 표준화된 파이프라인은 RNA-seq 분석의 재현성과 확장성 문제를 해결한다. Nextflow(Di Tommaso et al., 2017)와 Snakemake(Mölder et al., 2021)는 사전 정의된 분석 단계의 안정적 실행을 보장하고, nf-core/rnaseq(Ewels et al., 2020)와 같은 커뮤니티 큐레이션 파이프라인은 모범 사례 기본 매개변수를 제공한다. Galaxy(Afgan et al., 2018)와 iDEP(Ge et al., 2018) 같은 웹 기반 플랫폼은 접근 장벽을 낮추는 그래픽 인터페이스를 제공한다.

그러나 이러한 도구들은 근본적인 한계를 공유한다: 고정된 워크플로를 실행한다는 것이다. 분석 경로는 데이터를 검토하기 전에 결정되며, 예상치 못한 결과에 대한 적응은 수동 개입을 필요로 한다. 데이터셋이 예상보다 적은 DEGs를 산출할 때, 고정 파이프라인은 ORA를 계속 진행하며 — GSEA가 검출할 수 있는 생물학적으로 의미 있는 경로 수준의 변화를 놓칠 수 있다. 이러한 적응적 의사결정에 필요한 전문 지식은 전사체 분석의 주요 병목으로 남아 있다.

### 1.3 추론 엔진으로서의 대규모 언어 모델

대규모 언어 모델(LLMs)의 최근 발전은 코드 생성, 과학적 추론, 도메인 특이적 지식 합성에서 주목할 만한 능력을 보여주었다(Bubeck et al., 2023). 생물정보학에서 LLMs는 문헌 마이닝과 코드 지원에 적용되어 왔다. 그러나 이러한 응용은 LLM을 완전한 분석 워크플로를 탐색할 수 있는 자율 추론 에이전트가 아닌 수동적 도구로 취급한다.

계획, 실행, 적응하는 LLM 기반 에이전트의 개념은 소프트웨어 공학(Jimenez et al., 2024)과 과학적 발견(Boiko et al., 2023)에서 주목받고 있다. 그러나 우리가 아는 한, 전체 RNA-seq 분석 파이프라인에 LLM 기반 적응적 추론을 체계적으로 적용한 프레임워크는 없다.

### 1.4 본 연구의 기여

본 연구에서는 5가지 핵심 기여를 가진 오픈소스 프레임워크 ARIA를 제안한다:

1. **8개 의사결정 지점(DPs)으로 구성된 의사결정 프로토콜** — QC, DE 전략, 설계 인식, 시그니처 검출, 교차 검증, 해석, 민감도 분석, 보고서 생성을 포괄.
2. **적응적 전략 전환** — 중간 결과에 기반한 분석 방법 자동 전환.
3. **교차 방법 검증** — DESeq2, edgeR, limma-voom 간 결과 교차 검증.
4. **자동화된 맥락화** — LLM 기반 생물학적 해석.
5. **체계적 벤치마킹** — 다양한 난이도의 4개 공개 데이터셋에서 평가.

## 2. 방법

### 2.1 아키텍처

ARIA는 4개 계층으로 구성된다(Figure 1): (1) **추론 엔진(Reasoning Engine)** — 추론 백본으로 작동하는 LLM; (2) **의사결정 프로토콜(Decision Protocol)** — 규칙 기반 임계값과 LLM 추론을 결합한 8개의 공식화된 의사결정 지점; (3) **실행 엔진(Execution Engine)** — Docker 기반 R/Python 스크립트 생성 및 실행; (4) **분석 모듈(Analysis Modules)** — DESeq2, fgsea, STRING DB, WGCNA, 세포 유형 디컨볼루션, ssGSEA.

### 2.2 의사결정 프로토콜

각 의사결정 지점(DP)은 하이브리드 메커니즘으로 작동한다: 규칙 기반 임계값이 결정적 기준을 처리하고, LLM 추론 엔진이 맥락적 평가, 예외 상황, 해석적 작업을 처리한다.

| DP | 트리거 | 행동 | 검증 |
|----|--------|------|------|
| DP1 | QC 지표 확인 | 매핑률 ≥85%, rRNA <5%, 5'–3' bias 0.8–1.2 기반 판정 | 4개 전 벤치마크 |
| DP2 | DEG 수 평가 | ≥100: 표준; 10–100: GSEA 추가; <10: GSEA 우선 | SEQC, Airway, Fmr1 |
| DP3 | 메타데이터 검토 | Paired/blocked 설계 감지, 모델 수식 선택 | Airway(+21–47%), Pasilla(+4–30%) |
| DP4 | 예상 외 유전자 클러스터 | 플래그 및 세포 유형 디컨볼루션 추가 | 실제 사용에서 입증 |
| DP5 | 1차 DE 완료 | edgeR + limma-voom으로 교차 검증 | Fmr1(r=0.9999), Pasilla(r=0.9906) |
| DP6 | 경로 확인 | 문헌 기반 가설 생성 | 실제 사용에서 입증 |
| DP7 | 결과 확인 | LFC cutoff 민감도 분석 | 전 벤치마크 |
| DP8 | 분석 완료 | HTML 보고서 + Excel + 그림 생성 | 전 벤치마크 |

#### 2.2.1 DP5: 교차 방법 검증 — LLM의 역할

DP5에서 실행 엔진은 동일 데이터에 대해 DESeq2, edgeR(exact test 및 quasi-likelihood), limma-voom을 실행한다. LLM의 역할은 *일치도를 평가*하고 *불일치를 진단*하는 것이다:

1. **LLM 입력:** (a) 방법별 다중 cutoff에서의 DEG 수, (b) log2FC Pearson 상관, (c) 벤 다이어그램 중첩 통계를 포함하는 구조화된 요약.
2. **LLM 추론 과정:** *"이 교차 방법 결과를 바탕으로 방법 간 일치도를 평가하시오. DEG 수가 50% 이상 차이나거나 LFC 상관이 0.95 미만이면 이상치 방법을 식별하고 원인을 가설화하시오."*
3. **판단 출력:** 일치도가 높으면(r > 0.95, 중첩 > 70%) 1차 DESeq2 결과를 확인. 불일치 감지 시 원인 진단 보고서 생성.
4. **규칙 구성요소:** 상관 임계값(r > 0.95)과 중첩 임계값(>70%)은 규칙 기반; 진단적 해석은 LLM 기반.

#### 2.2.2 DP6: 문헌 기반 해석 — LLM의 역할

DP6은 가장 LLM 의존적인 의사결정 지점이다:

1. **LLM 입력:** (a) 상위 10개 DEG의 유전자명, fold change, 유의성 값; (b) GSEA 상위 농축 경로; (c) 실험 맥락(종, 조직, 조건).
2. **지식 출처:** LLM은 학습 cutoff까지의 생의학 문헌을 포괄하는 사전 학습된 지식 기반에서 정보를 추출한다. 이 단계에서 외부 데이터베이스 쿼리는 수행되지 않으며, LLM은 압축된 문헌 색인으로 기능한다.
3. **프롬프트 구조:** *"이 DEG [목록]과 농축 경로 [목록]이 [조직]의 [조건] 마우스에서 관찰되었다. (a) 상위 DEG의 알려진 기능, (b) 농축 경로가 이 모델의 알려진 생물학과 일치하는지, (c) 예상 외 발견에서 도출되는 검증 가능한 가설을 식별하시오. 모든 해석에 가설 수준이라는 라벨을 붙이시오."*
4. **출력:** 각 상위 DEG와 경로에 대한 구조화된 주석을 생성하며, 신뢰 수준을 명시적으로 표시.
5. **핵심 안전장치:** 모든 LLM 생성 해석에 면책 문구 태그: *"가설 수준 해석. 출판 전 인용을 독립적으로 검증하시오."*

#### 2.2.3 DP7: 민감도 분석 — LLM의 역할

DP7은 실행에서는 규칙 기반이지만 해석에서는 LLM의 도움을 받는다:

1. **실행(규칙 기반):** padj 임계값(0.01, 0.05, 0.1) × LFC 임계값(0, 0.5, 1.0, 2.0)의 조합 매트릭스를 자동 테스트하여 12셀 민감도 표를 생성.
2. **LLM 해석:** *"이 민감도 표를 분석하시오. 이 데이터셋에 대해 생물학적 유의성과 통계적 엄격성의 균형을 이루는 cutoff 조합을 식별하시오."*
3. **권장 출력:** 명시적 근거와 함께 최적 cutoff을 권장.

### 2.3 구현

ARIA는 Python 3.10+로 구현되었으며, R 분석 모듈은 Docker 컨테이너에서 실행된다. 핵심 도구: DESeq2 v1.46.0, edgeR, limma, fgsea, msigdbr, WGCNA, STRING DB v12. 모든 R 스크립트는 재현성을 위해 매개변수화된 템플릿에서 생성된다.

난이도는 세 가지 객관적 기준에 따라 분류하였다: (1) **예상 효과 크기**, (2) **설계 복잡성**, (3) **생물학적 변동성 대비 샘플 크기**.

### 2.5 평가 지표

(A) 알려진 생물학 대비 DEG 정확도(표적 유전자 복원율); (B) 의사결정 적절성(올바른 전략 선택); (C) 교차 방법 일치도(LFC 상관); (D) GSEA 경로 수율; (E) 분석 시간.

ARIA의 DP3은 paired 설계를 정확히 인식하고 naive `~ dex` 대신 `~ cell + dex`를 선택하였다.

**표 2. 설계 인식의 영향**

| Cutoff | Unpaired | Paired (ARIA) | 증가 |
|--------|----------|---------------|------|

7개의 알려진 dexamethasone 표적 유전자(DUSP1, KLF15, CRISPLD2, PER1, FKBP5, TSC22D3, ZBTB16) 전부가 높은 유의성(padj < 1e-19, LFC 2.6–7.4)으로 복원되었다.

### 3.3 벤치마크 간 알려진 표적 검증

**SEQC (UHRR vs HBRR):** 9/9 뇌 농축 유전자(GFAP, MBP, SYN1, GAD1, SLC17A7, NEFL, NEFM, NEFH, ENO2)가 HBRR에서 정확히 상향 발현으로 식별(LFC +2.2 ~ +12.7). 6/7 암 유전자(MYC, CCND1, CDK4, EGFR, ERBB2, TP53)가 UHRR에서 정확히 상향 발현으로 식별. BCL2는 알려진 뇌 발현과 일치하는 예상 외 방향을 보임.

**Fmr1 KO:** Fmr1 자체가 최상위 DEG로 검출(LFC = −0.98, padj = 3.7e-83). FMRP 조절 시냅스 단백질: Dlg4/PSD-95(LFC = −0.36), Shank3(LFC = −0.45), Camk2a(LFC = −0.42). NMDA 수용체 서브유닛: Grin1(LFC = −0.92), Grin2a(LFC = −0.81), Grin2b(LFC = −0.48). 모든 방향이 FMRP 매개 번역 조절 소실과 일치.

ARIA는 각 데이터셋의 난이도를 정확히 분류하였다:
- **SEQC (10,430 DEGs):** → "표준" (ORA + GSEA) ✓

### 3.6 DP3 + DP7: 공변량 감지 및 민감도 분석 (Pasilla)

Pasilla 데이터셋은 single-end와 paired-end 라이브러리가 혼합되어 기술적 공변량을 생성한다. ARIA의 DP3은 이 차이를 감지하고 공변량으로 포함(`~ type + condition`)하여 DEG 검출을 개선하였다:

| Cutoff | 공변량 없음 | 공변량 포함 (ARIA) | 증가 |
|--------|-----------|-----------------|------|
| padj<0.05, \|LFC\|>1.0 | 216 | **224** | **+4%** |
| padj<0.05, \|LFC\|>0.5 | 583 | **635** | **+9%** |
| padj<0.05, LFC 제한 없음 | 853 | **1,108** | **+30%** |

DP7(민감도 분석)은 12개 조합의 cutoff 표를 생성하여 결과 안정성에 대한 포괄적 관점을 제공하였다.

### 3.7 GSEA는 DEG 수를 넘어선 가치를 입증

풍부한 DEGs를 가진 데이터셋에서도 GSEA는 보완적 통찰을 제공하였다. Fmr1 KO에서 607개 GO:BP 경로가 농축(padj < 0.05)되어, 시냅스 신호전달, 번역 조절, 신경 발달의 전체적 교란을 보여주었다 — Fragile X 증후군 문헌에서 잘 문서화된 생물학적 과정들이다.

## 4. 논의

### 4.1 적응적 의사결정이 핵심 혁신

ARIA의 중심 기여는 확립된 분석 도구를 적응적으로 연결하는 추론 계층이다. Airway 벤치마크에서 설계 인식(DP3)은 DEG 검출을 21–47% 증가시켰다. 이는 naive 분석에서 놓쳤을 천 개 이상의 유전자를 나타내며 — 단일 적응적 의사결정에서 얻은 정량화 가능한 개선이다.

### 4.3 Human-in-the-Loop 설계

ARIA는 생물정보학자를 대체하지 않고 보강하도록 설계되었다. 생물학적 해석은 가설 수준으로 라벨링된다. 의사결정 로그는 감사와 검토를 가능하게 한다. 복잡한 상황은 전문가 판단을 위해 플래그된다.

### 4.4 한계점

#### 4.4.1 LLM 환각 위험과 완화

LLMs는 그럴듯하지만 사실적으로 부정확한 진술을 생성할 수 있다 — "환각(hallucination)"으로 알려진 현상이다. ARIA에서 이 위험은 DP6(문헌 기반 해석)에서 가장 두드러지며, LLM이 (a) 존재하지 않는 논문을 인용하거나, (b) 유전자에 부정확한 기능을 귀속하거나, (c) 메커니즘적 연결을 날조할 수 있다.

4가지 완화 전략을 구현한다:

**(1) 범위 제한:** LLM은 통계적으로 유의한 유전자와 경로만 해석하도록 요청받는다.

**(2) 명시적 라벨링:** 모든 LLM 생성 해석에 `[가설 — 독립적 검증 필요]` 태그를 부착한다. 의사결정 로그에 정확한 프롬프트와 응답이 기록되어 사후 감사가 가능하다.

**(3) 규칙 기반 출력과의 교차 참조:** LLM 해석은 통계 결과에 의해 제약된다. LLM이 상향 발현을 주장하지만 DE 결과가 하향 발현을 보이면 이 모순이 자동으로 플래그된다.

**(4) 인간 검토 체크포인트:** ARIA의 보고서에 "전문가 검토가 필요한 해석" 전용 섹션을 포함하여 도메인 전문가의 효율적 감사를 가능하게 한다.

**실제 관찰된 환각:** 실제 테스트에서 LLM이 간혹 유전자 기능을 경미하게 부정확하게 기술하거나(예: 단백질을 잘못된 세포 소기관에 귀속) 검증 불가능한 참고문헌을 한 번 생성한 것을 관찰하였다. 이러한 사례는 전문가 검토에서 발견되었으며, human-in-the-loop 설계의 필요성을 강화한다.

#### 4.4.2 통계적 파워

통계적 파워는 분석 방법이 아닌 샘플 크기에 의해 제약된다. ARIA는 이 한계를 투명하게 전달하고 전략을 적응시키지만(예: GSEA 전환), 소규모 실험에 내재된 근본적 통계적 제약을 극복할 수는 없다.

#### 4.4.3 세포 유형 혼동

Bulk RNA-seq는 세포 유형 내 유전자 발현 변화와 세포 유형 구성 변화를 구분할 수 없다. ARIA의 마커 기반 디컨볼루션(DP4)이 근사적 평가를 제공하지만, 확정적인 세포 유형 해상도 분석에는 single-cell 또는 single-nucleus RNA-seq가 필요하다.

#### 4.4.4 API 비용 및 재현성

ARIA는 LLM API 접근이 필요하다(분석당 ~$2–10). LLM 출력은 본질적으로 확률적이다; 통계 분석은 고정 시드를 사용하지만, 해석 구성요소(DP6)는 실행 간 약간 다를 수 있다. 의사결정 로그가 각 실행의 정확한 추론을 기록하여 이를 완화한다.

#### 4.4.5 범위

ARIA의 현재 버전은 bulk RNA-seq 2그룹 비교를 위해 설계되었다. 시계열 설계, single-cell RNA-seq, 다중 오믹스 통합으로의 확장이 계획되어 있으나 아직 구현되지 않았다.

### 4.5 향후 방향

Single-cell RNA-seq, 다중 오믹스 통합, 시계열 설계, 도메인 특화 미세 조정 LLM으로의 확장이 계획되어 있다.
