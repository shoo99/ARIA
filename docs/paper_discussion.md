# 4. Discussion

## 4.1 Adaptive Decision Making as the Core Innovation

The central contribution of ARIA is not the individual analysis tools — DESeq2, fgsea, and STRING DB are well-established — but the reasoning layer that connects them adaptively. Traditional pipelines treat RNA-seq analysis as a linear sequence of fixed steps. ARIA treats it as a decision tree where each node depends on the outcome of the previous step.

This distinction has practical consequences. In the Airway benchmark, ARIA's recognition of paired experimental design (DP3) increased DEG detection by 21-47% compared to a naive unpaired analysis. In datasets with few DEGs, ARIA's automatic switch from ORA to GSEA (DP2) enables pathway-level biological insights that a fixed ORA-only pipeline would miss entirely. These are not hypothetical scenarios — they reflect common situations encountered in real-world RNA-seq analysis where non-expert users routinely make suboptimal analytical choices.

The Decision Protocol formalizes what experienced bioinformaticians do implicitly: assess quality, evaluate results, adjust strategy, and validate findings. By encoding this reasoning process, ARIA makes it transparent, reproducible, and auditable through its decision log.

## 4.2 The Human-in-the-Loop Model

ARIA is designed as a human-in-the-loop system, not a fully autonomous replacement for bioinformaticians. This design choice is deliberate and reflects several considerations:

**Biological interpretation requires domain expertise.** While ARIA generates literature-contextualized hypotheses (DP6), these are explicitly labeled as hypothesis-level. The LLM may generate plausible but incorrect biological narratives, particularly for novel or poorly characterized systems. Expert review of interpretive claims remains essential.

**Edge cases require human judgment.** ARIA's Decision Protocol covers common scenarios, but unusual experimental designs, novel organisms, or unexpected technical artifacts may require decisions outside the protocol's scope. The framework is designed to flag uncertainty (via confidence scores) and defer to human judgment when appropriate.

**Reproducibility vs. non-determinism.** LLM outputs are inherently stochastic. While ARIA uses fixed random seeds for statistical analyses, the LLM reasoning component may produce slightly different interpretations across runs. The decision log and versioned Docker environments mitigate this, but users should be aware of this limitation.

It is important to note that ARIA does not replace nf-core/rnaseq for the upstream processing (alignment, quantification). Rather, ARIA operates on the downstream analysis: from count matrices through DE analysis, pathway enrichment, network analysis, and interpretation. The two tools are complementary.

## 4.4 Limitations

### 4.4.1 LLM Hallucination Risk
LLMs can generate plausible but factually incorrect statements, particularly when asked to cite specific papers or describe molecular mechanisms outside their training data. ARIA mitigates this by:
- Using rule-based thresholds for critical decisions (QC, DEG cutoffs)
- Labeling all interpretive content as hypothesis-level
- Providing the decision log for human audit

However, users should independently verify any literature citations or mechanistic claims generated by the LLM component.

### 4.4.2 Statistical Limitations
ARIA's statistical analyses are only as powerful as the underlying data. With small sample sizes (n=3 per group), DEG detection is inherently limited regardless of the analytical method. ARIA transparently communicates this limitation and adapts its strategy (e.g., switching to GSEA), but it cannot overcome fundamental statistical constraints.

### 4.4.3 Cell Type Confounding
Bulk RNA-seq cannot distinguish gene expression changes within a cell type from changes in cell type composition. ARIA's marker-based deconvolution (DP4) provides an approximate assessment, but single-cell or single-nucleus RNA-seq is required for definitive cell-type-resolved analysis.

### 4.4.4 Computational Cost
ARIA requires both computational resources (Docker, R/Python execution) and LLM API access. The API cost per analysis is approximately $2-10 depending on the dataset complexity and the extent of interpretive analysis. This may be a consideration for high-throughput applications.

### 4.4.5 Scope
The current version of ARIA is designed for bulk RNA-seq two-group comparisons (treatment vs. control). Extensions to time-series designs, single-cell RNA-seq, multi-omics integration, and non-model organisms are planned but not yet implemented.

## 4.5 Future Directions

**Multi-omics integration.** The adaptive reasoning framework naturally extends to integrating RNA-seq with ATAC-seq, proteomics, or metabolomics data. Each additional data type introduces new decision points (e.g., "Does the ATAC-seq accessibility change explain the observed DE?").

**Single-cell RNA-seq.** scRNA-seq analysis involves additional decisions about clustering resolution, cell type annotation, and trajectory inference. An scRNA-seq ARIA extension would address a significant need in the community.

**Domain-specific fine-tuning.** A fine-tuned LLM trained on bioinformatics literature and analysis logs could improve decision quality and reduce hallucination risk compared to general-purpose models.

**Community-contributed Decision Points.** The modular Decision Protocol design allows the community to contribute new DPs for specialized analyses (e.g., viral integration detection, allele-specific expression).

**Prospective validation.** While we benchmark ARIA retrospectively on published datasets, a prospective study comparing ARIA's recommendations with blinded expert analyses would provide stronger evidence of utility.
